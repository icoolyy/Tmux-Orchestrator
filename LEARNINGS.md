# 编排器学习笔记

## 2025-06-18 - 项目管理与代理监督

### 发现：网络搜索的重要性
- **问题**：开发者花费 2 小时以上尝试解决 Convex 中的 JWT 多行环境变量问题
- **错误**：作为 PM，我在用户提示之前没有建议网络搜索
- **教训**：在尝试 10 分钟失败后应该始终建议网络搜索
- **解决方案**：在全局 CLAUDE.md 中添加了"网络搜索是你的朋友"部分
- **影响**：网络搜索立即揭示了解决方案（用空格替换换行符）

### 洞察：仔细阅读错误信息
- **问题**：开发者在进行 base64 解码时浪费时间，而真正的错误是"缺少环境变量 JWT_PRIVATE_KEY"
- **教训**：在实现复杂解决方案之前始终验证实际的错误
- **模式**：开发者经常在不检查基本假设的情况下过度设计解决方案
- **PM 行动**：在批准解决方案之前询问"确切的错误信息是什么？"

### 项目经理最佳实践
- **坚定但建设性**：当开发者在没有文档的情况下编码时，必须坚持创建 LEARNINGS.md
- **状态报告**：直接的问题比开放式"进展如何？"获得更好的结果
- **升级时机**：如果 3 种方法失败，立即建议不同的策略
- **文档优先**：在继续编码之前强制执行文档，当卡住时

### 有效的通信模式
- **有效**："停止。给我状态：1) X 修复了？是/否 2) 当前错误？"
- **不太有效**："认证进展如何？"
- **关键**：具体的、编号的问题迫使清晰的回答

### 提醒系统
- **发现**：用户提醒我在结束对话之前设置签到提醒
- **实现**：使用 schedule_with_note.sh 配合具体的操作项
- **最佳实践**：始终用具体的下一步安排后续工作，而不是模糊的"检查进度"

## 2025-06-17 - 代理系统设计

### 多代理协调
- **挑战**：通信复杂度随着更多代理呈指数增长（n²）
- **解决方案**：以 PM 作为中央协调器的中心辐射模型
- **关键洞察**：结构化通信模板减少歧义和开销

### 代理生命周期管理
- **学习**：需要明确区分永久代理和临时代理
- **解决方案**：在终止代理之前实现适当的日志记录
- **目录结构**：agent_logs/permanent/ 和 agent_logs/temporary/

### 质量保证
- **原则**：PM 必须"对测试和验证一丝不苟"
- **实现**：验证清单，没有捷径，跟踪技术债务
- **关键**：信任但要验证 - 始终检查实际实现

## 常见陷阱

1. **不使用可用工具**：网络搜索、文档、社区资源
2. **循环解决问题**：反复尝试相同的方法而不退后一步
3. **缺少上下文**：不检查其他 tmux 窗口中的错误详情
4. **时间管理不当**：不为调试尝试设置时间限制
5. **不完整的交接**：不为未来的代理记录解决方案

## 编排器特定洞察

- **保持高层**：不要陷入实现细节
- **模式识别**：跨项目的类似问题（认证、环境变量等）
- **跨项目知识**：使用一个项目的洞察来帮助另一个项目
- **主动监控**：检查多个窗口以尽早发现问题

## 2025-06-18 - 后续会话 - 认证成功故事

### 有效的 PM 干预
- **情况**：开发者在 JWT 认证上挣扎了 3 小时以上
- **关键行动**：当我看到错误已解决时发送了直接鼓励
- **结果**：激励开发者正确记录学习心得
- **教训**：及时的正面反馈与纠正指导同样重要

### 跨窗口智能
- **发现**：可以在开发者工作时监控服务器日志
- **应用**：在开发者注意到之前就看到了 JWT_PRIVATE_KEY 错误已解决
- **价值**：基于实时监控的主动鼓励
- **最佳实践**：始终检查相关窗口（服务器、日志）以获取上下文

### 文档强制执行
- **挑战**：开发者通常在解决方案有效时跳过文档
- **解决方案**：发送关于要记录什么的特定提醒
- **示例**：列出了 LEARNINGS.md 中要包含的确切项目
- **影响**：确保机构知识被捕获

### Claude 计划模式发现
- **功能**：Claude 有一个通过 Shift+Tab+Tab 激活的计划模式
- **关键序列**：按住 Shift，按 Tab，再按 Tab，释放 Shift
- **关键步骤**：必须验证"⏸ plan mode on"出现 - 可能需要多次尝试
- **Tmux 实现**：`tmux send-keys -t session:window S-Tab S-Tab`
- **验证**：`tmux capture-pane | grep "plan mode on"`
- **故障排除**：如果未激活，发送额外的 S-Tab 直到确认
- **用户纠正**：用户最初不得不手动为我激活它
- **用例**：为复杂的密码重置实现激活了计划模式
- **最佳实践**：在发送规划请求之前始终验证激活
- **关键学习**：计划模式在编码开始之前强制执行深思熟虑的方法
